.NET Framework类所在名称空间

名称空间 说 明
System-Collections 这是集合的根名称空间。子名称空间也包含集合，如System.Collections.Concurrent和 System.Collections. Generic
Systern.Data 这是访问数据库的名称空间。System.Data.SqlClient包含访问SQL Server的类
System.Diagnostics 这是诊断信息的根名称空间，如事件记录和跟踪(在System.Diagnostics.Tracing名称空间中）
System.Globalization 该名称空间包含的类用于全球化和本地化应用程序
System.10 这是文件10的名称空间，其中的类访问文件和目录，包括读取器、写入器和流
System.Net 这是核心网络的名称空间，比如访问DNS服务器，用System.Net.Sockets创建套接字
System.Threading 这是线程和任务的根名称空间。任务在System.Threading.Tasks中定义
System.Web 这是ASP.NET的根名称空间。在这个名称空间下面定义了许多子名称空间，如System.Web.UI、 System.Web.ULWebControls 和 System.Web.Hosting
System. Windows 这是用于带有WPF的Windows桌面应用程序的根名称空间。子名称空间有 System.Windows.Shapes、System.Windows.Data 和 System.Windows.Documents

.NET Core

.NET Core是新的开源.NET框架，运行库是CoreCLR库，包含集合类的框架、文件系统、控制台和XML都在CoreFX库中。.NET Core采用模块化设计，框架和应用程序一起交付，应用程序发布时与它需要的框架一起打包。它使用新的命令行程序.NET Core Command Line (CLR)。

程序集（Assembly）

.NET程序的库和可执行文件称为程序集（Assembly）。Assembly是预编译好的、完全自描述性的逻辑单元，它可以存储在多个文件中（动态程序集存储在内存中）。存储在多个文件中的程序集需要一个包含入口点的主文件来描述其他文件。可执行的程序集包含一个主程序入口点，库程序集不包含。

C#基本语法

  • 语句以分号;结尾，以花括号{}组合为块。
  • 单行注释//，多行注释/**/，///三斜杠用于生成XML说明文档。
  • 区分大小写。
  • 变量必须进行初始化。类和结构中的变量的默认值为0，方法中的局部变量必须进行显式初始化。
  • 可以通过var关键字进行类型推断。
  • 数据类型分为值类型和引用类型。

类的成员

成 员 说 明
字段 字段是类的数据成员，它是类型的一个变量，该类型是类的一个成员
常量 常量与类相关(尽管它们没有static修饰符)。编译器使用真实值代替常量
方法 方法是与特定类相关联的函数
属性 属性是可以从客户端访问的函数组，其访问方式与访问类的公共字段类ftt。C#为读写类中的属性提供了 专用语法，所以不必使用那些名称中嵌有Get或Set的方法。因为属性的这种语法不同于一般函数的语法， 所有在客户端代码中，虚拟的对象被当作实际的东西
构造函数 构造函数是在实例化对象时自动调用的特殊函数。它们必须与所属的类同名，且不能有返回类型。构造 函数用于初始化字段的值
索引器 索引器允许对象用访问数组的方式访问。索引器参见第8章
运算符 运算符执行的最简单的操作就是加法和减法。在两个整数相加时，严格地说，就是对整数使用“+”运算 符。C#还允许指定把已有的运算符应用于自己的类(运算符重载)。第8章将详细论述运算符
事件 事件是类的成员，在发生某些行为(如修改类的字段或属性，或者进行了某种形式的用户交互操作)时，它可 以让对象通知调用方。客户可以包含所谓“事件处理程序”的代码来响应该事件。第9章将详细介绍事件
析构函数 析构函数或终结器的语法类似于构造函数的语法，但是在CLR检测到不再需要某个对象时调用它。它们 的名称与类相同，但前面有一个“〜”符号。不可能预测什么时候调用终结器。终结器详见第5章
类型 类可以包含内部类。如果内部类型只和外部类型结合使用，就很有趣

结构

如果只需要一个小的数据结构，由于性能原因，我们最好使用结构，只需用关键字struct代替class。

结构与类的区别：

  • 结构是值引用，遵循其他数据类型所遵循的规则。
  • 不能从结构中继承。
  • 结构的构造函数与类的构造函数相同。
  • 通过添加ref修饰符，可以按引用传递结构。而给类使用ref，则是传递引用的引用。（ref关键字在声明和调用方法时都需要提供）

使用out关键字，可以使方法返回多个类型不同的关键字。同样需要在声明和调用时都提供。

可空类型

可以在任意结构类型后面添加“?”声明可空类型。

枚举类型

Object类的方法

所有.NET类都派生自SystemObject，所以我们可以访问由Object类定义的成员方法：

  • ToString()：获取对象的字符串表示，没有提供格式化选项。
  • GetHashCode()：用于位于散列表或字典中的对象。
  • Equals(),ReferenceEquals()：有三个比较对象相等性的方法。
  • Finalize()：用于垃圾回收。
  • GetType()：提供对象成员所属类的更多信息。
  • MemberwiseClone()：复制对象，并返回对副本的一个引用。

继承

继承的类型以及在C#中的支持情况：

  • 单重继承：一个类派生自一个基类。C#采用此种继承。
  • 多重继承：一个类派生自多个类。C#不支持。
  • 多层继承：B派生自A，C派生自B。C#支持。
  • 接口继承：C#允许接口多重继承。

虚方法

使用virtual关键字定义的虚方法可以在派生类中被重写，重写方法时需要使用override关键字。

多态性

如果一个方法的参数是一个类，那么它的派生类也可以传递给该方法。

抽象类和抽象方法

使用abstract声明抽象类和抽象方法。抽象类不能实例化，抽象方法不能直接实现，必须在非抽象派生类中重写。

如果类包含抽象方法，则该类也必须声明为抽象的。

密封类和密封方法

使用sealed修饰符声明的类不能创建子类，声明的方法不能被重写。

派生类的构造函数

构造函数总是按照层次结构的顺序，从System.Object开始，从上向下调用，直到要实例化的类为止。

访问修饰符

修饰符 应用于 说 明
public 所有类型或成员 任何代码均可以访问该项
protected 类型和内嵌类型的所有成员 只有派生的类型能访问该项
internal 所有类型或成员 只能在包含它的程序集中访问该项
private 类型和内嵌类型的所有成员 只能在它所属的类型中访问该项
protected internal 类型和内嵌类型的所有成员 只能在包含它的程序集和派生类型的任何代码中访问该项

对于嵌套类型，内部类总是可以访问外部类的所有成员，包括私有成员。

接口

  • 接口包含方法、属性、索引器和事件的声明，但是不能提供其中任何成员的实现方式。
  • 不能实例化接口，但是可以声明接口类型的变量。
  • 不能声明成员的修饰符。

as和is运算符

用于对象类型判断和转换的两个操作符：

  • as返回对象的引用，但是从不抛出异常，如果对象不是所要求的类型，就返回null。
  • is判断对象是否是指定的类型，返回true或者false。
